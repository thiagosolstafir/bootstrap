Escala tipográfica

Todo o grid de tipografia é baseado em duas variáveis Less no nosso arquivo variables.less: @baseFontSize e @baseLineHeight. O primeiro é o tamanho de fonte base usado em todo projeto e o segundo é a propriedade line-height base

Nós usamos estas variáveis, e alguma matemática, para criar as margens, paddings, e line-heights de todos os tipos e mais

Código blocos de código inline e em bloco

Inline

Wrap inline snippets of code with <code>.

Por exemplo, a tag <code>section</code> deve ser considerada como inline.

Basic block

Use <pre> para múltiplas linhas de código. Tenha certeza de que você escapou qualquer entidade HTML para renderizar corretamente

<p>Algum texto aqui...</p>

<pre>
  &lt;p&gt;Algum texto aqui...&lt;/p&gt;
</pre>

Nota: Tenha certeza de manter o código entre as tags <pre> a extrema esquerda, pois a tag conserva os tabs.

Você pode adicionar uma classe .pre-scrollable que irá setar o max-height de 350px para prover um scrollbar no eixo Y.

Google Prettify

Take the same <pre> element and add two optional classes for enhanced rendering.

<p>Algum texto aqui...</p>

<pre class="prettyprint
     linenums">
  &lt;p&gt;Algum texto aqui...&lt;/p&gt;
</pre>

Faça o Download do google-code-prettify e leia o readme para saber como usar.

Formulários

HTML e CSS flexíveis

A melhor parte bout forms no Bootstrap é que todos os inputs e controles possuem uma ótima aparência não importa como você construa o markup. Nenhum HTML supérfulo é requerido, mas nós podemos provê os padrões onde é necessário

Layouts mais complicados vem com classes escaláveis e sucintas para fácil estilização e manipulação de eventos, então é coberta em cada passo

Quatro layouts inclusos

O Bootstrap vem com suporte a 4 tipos de layouts para os formulários

  • Vertical (padrão)
  • Busca
  • Inline
  • Horizontal

Diferentes tipos de layout para formulários requer alguma mudança de marcação, mas os controles se comportam da mesma maneira

Controles de estados e mais

Os estilos de formulários do Bootstrap para todas as bases de controles de formulário como input, textarea, e select como você espera. Mas ele também vem com um número de componentes customizados como inputs do tipo appended e prepended e suporte para lista de checkboxes

Estados como erro, aviso, e sucesso são incluídos para cada tipo de controle de formulário. Também está incluído estilos para controles desabilitados

Quatro tipo de forms

O Bootstrap provê marcação simples e estilos para quatro estilos comuns de formulários web

Nome Classe Descrição
Vertical (padrão) .form-vertical (não requerido) Estagnado, alinhado a esquerda sob os controles
Inline .form-inline Rótulo alinhado a esquerda e controles inline-block para estilo compacto
Busca .form-search Texto com arredondamento extra para uma busca títpica estética
Horizontal .form-horizontal Flutuando a esquerda, alinhado a direita com rótulo na mesma linha dos controles

Forms de exemplo usando simplesmente controles de formulários, sem marcação extra

Formulário básico

Com o 2.0, nós temos um padrão inteligente e conciso para formulários padrões. Sem marcação extra, somente controle de formulários

Associated help text!

Exemplo de texto de ajuda em bloco

<form class="well">
  <label>Nome do rótulo</label>
  <input type="text" class="span3" placeholder="Digite algo">
  <span class="help-inline">Associated help text!</span>
  <label class="checkbox">
    <input type="checkbox"> Oha eu aqui
  </label>
  <button type="submit" class="btn">Enviar</button>
</form>

Formulário de busca

Refletindo padrões do estilo WebKit, adicione a classe .form-search para bordas arredondadas extras

<form class="well form-search">
  <input type="text" class="input-medium search-query">
  <button type="submit" class="btn">Busca</button>
</form>

Form inline

Inputs são elementos blocos para começar. Para .form-inline e .form-horizontal, nós usamos inline-block

<form class="well form-inline">
  <input type="text" class="input-small" placeholder="Email">
  <input type="password" class="input-small" placeholder="Senha">
  <label class="checkbox">
    <input type="checkbox"> Lembrar de mim
  </label>

Buttons for actions

As a convention, buttons should only be used for actions while hyperlinks are to be used for objects. For instance, "Download" should be a button while "recent activity" should be a link.

Button styles can be applied to anything with the .btn class applied. However, typically you'll want to apply these to only <a> and <button> elements.

Cross browser compatibility

IE9 doesn't crop background gradients on rounded corners, so we remove it. Related, IE9 jankifies disabled button elements, rendering text gray with a nasty text-shadow that we cannot fix.

Multiple sizes

Fancy larger or smaller buttons? Add .btn-large, .btn-small, or .btn-mini for two additional sizes.


Disabled state

For disabled buttons, add the .disabled class to links and the disabled attribute for <button> elements.

Primary link Link

Heads up! We use .disabled as a utility class here, similar to the common .active class, so no prefix is required.

One class, multiple tags

Built as a sprite

Instead of making every icon an extra request, we've compiled them into a sprite—a bunch of images in one file that uses CSS to position the images with background-position. This is the same method we use on Twitter.com and it has worked well for us.

All icons classes are prefixed with .icon- for proper namespacing and scoping, much like our other components. This will help avoid conflicts with other tools.

Glyphicons has granted us use of the Halflings set in our open-source toolkit so long as we provide a link and credit here in the docs. Please consider doing the same in your projects.

How to use

Bootstrap uses an <i> tag for all icons, but they have no case class—only a shared prefix. To use, place the following code just about anywhere:

<i class="icon-search"></i>

There are also styles available for inverted (white) icons, made ready with one extra class:

<i class="icon-search icon-white"></i>

There are 120 classes to choose from for your icons. Just add an <i> tag with the right classes and you're set. You can find the full list in sprites.less or right here in this document.

Heads up! When using beside strings of text, as in buttons or nav links, be sure to leave a space after the <i> tag for proper spacing.

Use cases

Icons are great, but where would one use them? Here are a few ideas:

  • As visuals for your sidebar navigation
  • For a purely icon-driven navigation
  • For buttons to help convey the meaning of an action
  • With links to share context on a user's destination

Essentially, anywhere you can put an <i> tag, you can put an icon.

Examples

Use them in buttons, button groups for a toolbar, navigation, or prepended form inputs.
