Estilos globais para reset CSS, tipografia e links

Requer o doctype de HTML5

O Bootstrap faz o uso de elmentos HTML e propriedades CSS que requer o uso de um doctype HTML5. Tenha certeza de inclir isto em cada página Bootstratiada em seu projeto

<!DOCTYPE html>
<html lang="en">
  ...
</html>

Tipografia e links

Com o arquivo scaffolding.less, nós escolhemos uma exibição global para tipografia e links. Especificadamente, nós:

  • Removemos as margens do body
  • Escolhemos o background-color: white; no body
  • Usamos o @baseFontFamily, @baseFontSize, e @baseLineHeight atributos para a tipografia básica
  • Escolhemos cores globais de link via @linkColor e aplicamos linha abaixo do link com :hover

Reset via Normalize

Com o Bootstrap 2, o reset tradicional de elementos foi feito via Normalize.css, um projeto do Nicolas Gallagher que também usou o poder do HTML5 Boilerplate.

O novo reset pode ainda ser encontrado no reset.less, mas com muitos elementos removidos para brevidade e eficácia

O sistema de grid padrão do provido pelo Bootstrap utiliza 12 columns que renderiza uma largura de 724px, 940px (padrão sem css responsivo incluído), e 1170px. Abaixo de resoluções 767px, as colunas se tornam fluidas e estagnadas verticalmente.

Aninhando colunas

Com o sistema de grid estático(não fluido) no Bootstrap, aninhamento é fácil. Para aninhar seu conteúdo, simplesmente adicione um novo .row e as colunas em .span* colunas de uma classe existente .span*.

Exemplo

Linhas aninhadas devem incluir uma coleção de colunas que adiciona o número do seu pai. Por exemplo, duas colunas aninhadas .span3 devem ser colocadas dentro de uma .span6.

Nível 1 da coluna
Nível 2
Nível 2 
<div class="row">
  <div class="span6">
    Coluna nível 1
    <div class="row">
      <div class="span3">Nível 2</div>
      <div class="span3">Nível 2</div>
    </div>
  </div>
</div>

Porcento, não pixels

O sistema de grid fluid usa porcento para largura das colunas ao invés de pixels fixo. Ele também a mesma variação responsive como o sistema de grid fixo, certificando de que as proporções permanecerão em resoluções diferentes e dispositivos

Linhas fluidas

Qualquer linha fluida pode ser modificada mudando .row para .row-fluid. A coluna fica da mesma maneira, sendo super direto trocar entre layout fixos e fluidos

Marcação

<div class="row-fluid">
  <div class="span4">...</div>
  <div class="span8">...</div>
</div>

Teste de fluido

Aninhando com grids fluids é um pouco diferente: o número de colunas aninhadas não precisa ser igual ao pai. Ao invés disto, suas colunas são reiniciadas em cada nível por cada linha tem 100% da sua coluna pai.

Customização do grid

Variável Valor padrão Descrição
@gridColumns 12 Número de colunas
@gridColumnWidth 60px Largura da coluna
@gridGutterWidth 20px Espaço negativo entre colunas

Variáveis no LESS

Construído dentro do Bootstrap uma útil coleção de variáveis para customizar o grid system de 940px, documentado acima. Todas variáveis para grid estão armazenados em uma variável em variables.less

Como customizar

Modificar o grid significa mudar as três variáveis @grid* e recompilar o código. Mudando as variáveis de grid em variables.less e usá-los em uma das quatro maneiras de recompilar. Se você está adicionando mais colunas, tenha certeza de adicionar o CSS para elas em grid.less

Continue responsivo

Customização do grid só funciona no nível padrão, no grid de 940px. Para manter o aspecto do Bootstrap responsivo, nós também temos uma customização do grid em responsive.less

Layouts templates básicos para criar páginas

Layout fixo

O layout padrão e simples de 940px de largura e centralizado para qualquer site ou página com um único <div class="container">

<body>
  <div class="container">
    ...
  </div>
</body>

Layout fluido

<div class="container-fluid"> te dá uma estrutura de página flexível, com largura máxima e mínima, e com uma barra lateral direita. Ótima para aplicativos e documentação

<div class="container-fluid">
  <div class="row-fluid">
    <div class="span2">
      <!--conteúdo da lateral-->
    </div>
    <div class="span10">
      <!--conteúdo do corpo-->
    </div>
  </div>
</div>

Design responsivo Media queries para vários dispositivos e resoluções

O que eles fazem

Os Media queries permitem ter CSS customizado baseado em um número de condições—aspecto, larguras, dipo de exibição, etc—mas geralmente focado em min-width and max-width.

  • Modificar a largura da coluna no seu grid
  • Elementos estagnados ao invés de float quando for necessário
  • Redimensionando cabeçalhos e textos para ser mais apropriados para dispositivos

Usar media queries responsivamente e somente como princípio para a audiência de dispositivos móveis. Para projetos mariores, considere dedicar bases de código e não em camadas de media queries

Dispositivos suportados

O Bootstrap suporta uma utilidade de media queries em um único arquivo para ajudar a fazer seus projetos mais apropriados para diferentes dispositivos e resoluções de tela. Aqui está o que é incluso:

Nome Largura do layout largura da coluna Largura do espaçamento
Smartphones 480px and below Colunas fluidas, larguras não fixas
Smartphones e tablets 767px and below Colunas fluidas, larguras não fixas
Tablets no formato na orientação horizontal 768px and above 42px 20px
Padrão 980px and up 60px 20px
Resolução de tela grande 1200px and up 70px 30px

Requer meta tag

Para certificar-se que os dispositivos responsivos são exibidos apropriadamente, inclua a meta tag viewport

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Usando media queries

O Bootstrap não automaticamente inclui estes media queries, mas entendê-lo e adicioná-los é muito fácil e requer a mínima configuração. Você tem algumas opções, para incluir funcionalidades responsivas do Bootstrap

  1. Use a versão responsive compilada, o bootstrap-responsive.css
  2. Adicione @import "responsive.less" e recompile o Bootstrap
  3. Modifique e recompile responsive.less como um arquivo separado

Por que não simplesmente incluí-lo? Verdade seja dita, nem tudo precisa ser responsivo. Ao invés de encorajar desenvolvedores a remover esta funcionalidade, nós configuramos para melhor habilitá-la

  /* dispositivo no formato paisagem e abaixos */
  @media (max-width: 480px) { ... }

  /* dispositivo em paisagem até os tablets em formato porta-retrato */
  @media (max-width: 767px) { ... }

  /* tables em formato porta retrato até os desktos no formato paisagem */
  @media (min-width: 768px) and (max-width: 979px) { ... }

  /* Desktop maiores */
  @media (min-width: 1200px) { ... }

Classes utilitárias responsivas

O que elas são

Para desenvolvimento rápido em dispositivos móveis, uses estas classes utilitárias básicas que mostra e esconde conteúdo por dispositivo

Quando usar

Use uma limitada e básica versão do seu site para evitar criar diferentes versões do mesmo site. Ao invés disto, use-os como complemento de cada apresentação do dispositivo

Por exemplo, você pode mostrar um elemento <select> para navegação em layouts mobile, mas não em tablets ou desktops
